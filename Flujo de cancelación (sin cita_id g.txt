Flujo de cancelación (sin cita_id guardado)
Paso 1 — Buscar las reservas activas del cliente por teléfono

GET /api/v1/agenda/reservas?telefono=+595971450246
X-Tenant-Key: ak_live_...
X-Tenant-ID: studioferreira
X-Branch-ID: 1

  "data": {

    "total": 2,

El campo cancelable: true/false ya le dice al bot cuáles puede ofrecer para cancelar, sin necesidad de lógica extra.

Paso 2 — Mostrar al usuario y confirmar

El bot presenta las citas con cancelable: true y pregunta cuál desea cancelar.

Paso 3 — Cancelar

DELETE /api/v1/agenda/reservas/42
X-Tenant-Key: ak_live_...
X-Branch-ID: 1
{ "motivo": "No puedo ir ese día" }
Parámetros opcionales de la búsqueda:

Parámetro	Por defecto	Descripción
telefono	—	Número del cliente (más útil para bots)
email	—	Email del cliente
cliente_id	—	ID interno
estado	activas	activas, pendiente, confirmada, realizada, cancelada, todos
fecha_desde	hoy	Solo reservas desde esta fecha
fecha_hasta	+60 días	Solo reservas hasta esta fecha
